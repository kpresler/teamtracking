Welcome to the Software Engineering Team Tracking Application!

The purpose of this application is to support tracking student teams in software engineering courses (or any other courses that feature team-based projects) so that the teaching staff can observe and track team challenges, and offer support to teams in overcoming these challenges. This application is built around the Team Collaboration Reflection Survey (TCRS) from SIGCSE 2022
You must first log in, or all API actions will fail.
After loggin in, two sets of data must be initialised:
  • Initialise the questions used in your version of the TCRS. Each question has a question text, which must exactly match against that in the CSV file you will submit later on. Additionally, questions have a type -- positive questions are ones where we would expect a successful team to answer "agree" or "strongly agree"; negative questions are ones where would expect successful teams to answer "disagree" or "strongly disagree". text questions are natural-language processed, and other questions are skipped when scoring the survey (but are still available for viewing later on).
  • Initialise the iterations in your project. The idea here is that we want to track team challenges and how they are resolved, or get worse, week after week. To display meaningful reports, the system needs to know how many iterations your project has, and the order in which they are arranged. The displayed value is just that -- the value you want displayed to the user(s) of this system (TAs, etc). This is designed to be human-readable, for instance "Iteration A", "Iteration B", "Iteration 3", etc. Make sure whatever values you provide here match against what is present in your version of the TCRS -- lookup is exact and any differences will cause it to fail. The sequential value is used for tracking their order internally, and can be either 0-indexed or 1-indexed, but cannot contain any gaps.
Next, load in your TCRS submissions from a CSV file.
You can then view a summary of teams for a given iteration, and (linked) details for each team.
For a reference of the API and to view stored data, check out the API.
